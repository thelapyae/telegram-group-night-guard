# Security Policy

## Reporting a vulnerability

Please report security issues privately through GitHub's security advisory
feature. Do not open a public issue containing a bot token, group configuration,
server address, or other credential.

Community reports are untrusted input. Permanent bans require a current Telegram
administrator decision; report thresholds only trigger review and temporary mute.

## Exposed bot tokens

If a token is exposed, revoke it immediately with [@BotFather](https://t.me/BotFather),
replace the value in the deployment's `.env`, and restart or rerun the bot.
